# Security Policy

## Supported Scope

Security reports are welcome for:

- Firmware (`firmware/esp`)
- Control panel (`apps/control-panel`)
- Shared core and simulator integration (`packages/lightpath`, `apps/simulator`)

## Reporting a Vulnerability

Please report vulnerabilities privately. Preferred method:

1. Use the repository host's private vulnerability reporting flow (for example, GitHub Security Advisories).
2. Include:
   - affected component/path
   - reproduction steps or proof of concept
   - impact assessment
   - suggested mitigation (if available)

Do not open public issues for unpatched security vulnerabilities.

## Response Goals

- Initial acknowledgement within 5 business days.
- Triage and remediation timeline based on severity and exploitability.
